# Flutter/Dart 文档本地化工具 - 站点镜像服务

## 双阶段镜像流程（单按钮触发）

**目标**：
- 先提升下载吞吐，再批量本地化，避免"下载被改写拖死"。
- **视觉一致性**：确保镜像站点与原站在视觉上完全一致，包括样式、布局、字体、图标等。
- 所有数据目录需要在【工程目录】（定义见 [flutter_doc_constraints 名词约束](file:///c:/app/flutter_doc/.trae/skills/flutter_doc_constraints/SKILL.md#名词约束)）目录下data目录下，详细要求见 [flutter_doc_constraints 工程约束](file:///c:/app/flutter_doc/.trae/skills/flutter_doc_constraints/SKILL.md#工程约束)。

### GUI 触发补充

- “开始镜像”按钮必须真正触发镜像服务，不能只写一条占位日志
- GUI 层负责创建后台 worker 与 `QThread`，服务层继续复用 `MirrorService`
- 日志、完成态、失败态通过信号槽返回主线程
- 阶段进度优先从 SQLite `task_status` 读取，保证界面与实际任务一致
- `task_status.total_count` 必须由 `url_queue` / `localize_queue` 聚合得出，不能直接用临时处理计数回填
- 下载阶段终态和本地化阶段终态要分别结算，避免下载完成后界面仍显示“运行中”
- 关闭窗口时应协作式停止任务，避免线程和数据库连接遗留

## 阶段 1：原站镜像下载

### 1. 初始化

- 创建 `raw mirror` 目录结构
- 检查数据库中是否存在未完成任务
- 初始化下载线程池

### 2. 种子 URL 发现

| 站点 | 种子来源 |
|------|---------|
| `docs.flutter.dev` | `sitemap.xml` |
| `dart.dev` | `sitemap.xml` |
| `api.flutter.dev` | `index.html` + `flutter/index.json` + 同域递归 |
| `api.dart.dev` | `index.json` + 同域递归 |

### 3. 递归发现链接

解析已下载页面中的所有站内资源链接：

| 资源类型 | 解析方式 |
|---------|---------|
| HTML | `href`/`src`/`srcset` 属性 |
| CSS | `url()` 函数 |
| JS/JSON | 可识别的站内 URL 模式 |

**覆盖资源类型**：manifest、opensearch、xml、json、pdf、font、image、js、css 等

### 4. 下载执行

```
┌─────────────────────────────────────────────────────────────┐
│                     下载执行流程                             │
│                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│  │ 从队列  │ →  │ 检查是  │ →  │ 下载到  │ →  │ 更新    │ │
│  │ 获取URL │    │ 否已下载│    │ raw     │    │ 数据库  │ │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘ │
│                      ↓ 已存在                               │
│                 ┌─────────┐                                 │
│                 │ 跳过    │                                 │
│                 └─────────┘                                 │
└─────────────────────────────────────────────────────────────┘
```

**下载规则**：

| 数据库记录 | 文件状态 | 操作 |
|-----------|---------|------|
| 不存在 | - | 重新发现并下载 |
| 存在 | 存在 | 跳过 |
| 存在 | 不存在 | 重新下载 |

**下载特性**：
- 线程池并发下载（`download_concurrency` 控制）
- 大文件使用 `.part` 临时文件 + Range 续传
- 资源写入 `raw mirror` 目录
- 元数据写入 `resource_cache` 表

**状态流转**：`queued` → `downloading` → `downloaded` / `failed`

### 5. 静态资源下载（视觉一致性保证）

为确保镜像站点与原站视觉完全一致，必须下载以下所有静态资源：

#### 样式资源

| 资源类型 | 文件扩展名 | 说明 |
|---------|-----------|------|
| CSS 样式表 | `.css` | 页面布局、颜色、字体样式 |
| CSS 预处理器 | `.scss`, `.sass`, `.less` | 如有暴露的源文件 |
| 内联样式 | `<style>` 标签 | HTML 内嵌样式 |

#### 字体资源

| 资源类型 | 文件扩展名 | 说明 |
|---------|-----------|------|
| Web 字体 | `.woff`, `.woff2`, `.ttf`, `.otf`, `.eot` | 自定义字体 |
| 图标字体 | `.woff`, `.woff2` | Material Icons、FontAwesome 等 |

#### 图片资源

| 资源类型 | 文件扩展名 | 说明 |
|---------|-----------|------|
| 位图 | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.bmp` | 图片、图标、背景 |
| 矢量图 | `.svg` | 图标、Logo、矢量图形 |
| 图标 | `.ico` | 网站图标 |

#### 脚本资源

| 资源类型 | 文件扩展名 | 说明 |
|---------|-----------|------|
| JavaScript | `.js` | 页面交互、动态功能 |
| JavaScript 模块 | `.mjs` | ES 模块 |
| TypeScript | `.ts` | 如有暴露的源文件 |

#### 其他资源

| 资源类型 | 文件扩展名 | 说明 |
|---------|-----------|------|
| Web Manifest | `.webmanifest` | PWA 配置 |
| 站点地图 | `.xml` | sitemap |
| 配置文件 | `.json` | 各种配置 |
| PDF 文档 | `.pdf` | 文档资源 |
| 视频资源 | `.mp4`, `.webm` | 视频内容 |
| 音频资源 | `.mp3`, `.ogg`, `.wav` | 音频内容 |

#### 外域静态资源处理

对于页面依赖的外域静态资源（如 CDN 资源），需要特殊处理：

| 资源来源 | 处理方式 |
|---------|---------|
| 四站互链资源 | 下载并本地化 |
| 外域 CDN 资源 | 下载到本地，改写链接 |
| 外域字体 | 下载到本地，改写 CSS |
| 外域图片 | 下载到本地，改写链接 |

### 6. 阶段 1 完成判定

- `url_queue` 中所有 URL 状态为 `downloaded` 或 `failed`
- 所有静态资源（CSS、JS、字体、图片）已下载完成
- 统计成功/失败数量，记录日志

---

## 阶段 2：本地化重建

### 1. 本地化队列生成

- 将 `url_queue` 中 `downloaded` 的 URL 写入 `localize_queue`
- 状态初始化为 `queued`

### 2. HTML/CSS 改写

```
┌─────────────────────────────────────────────────────────────┐
│                   HTML/CSS 改写流程                          │
│                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│  │ 读取    │ →  │ 保护    │ →  │ 改写    │ →  │ 恢复    │ │
│  │ raw文件 │    │ 代码块  │    │ 链接    │    │ 占位符  │ │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘ │
│                      ↓                                       │
│                 ┌─────────┐                                 │
│                 │ 写入    │                                 │
│                 │ localized│                                │
│                 └─────────┘                                 │
└─────────────────────────────────────────────────────────────┘
```

**改写规则**：
- **路径处理**：严格执行相对路径，不使用域名或 IP
- **站内链接**：改写为本地路径
- **四站互链**：同步本地化处理
- **外域链接**：保留原链接不变

**代码块保护**：
- 生成占位符替换代码块/签名
- 改写完成后恢复占位符

**CSS 改写规则**：
- 改写 CSS 中 `url()` 引用的资源路径
- 保持 CSS 选择器和样式规则不变
- 确保字体引用正确指向本地字体文件

### 3. 资源同步（视觉一致性关键）

| 方式 | 优先级 | 说明 |
|------|-------|------|
| 硬链接 | 优先 | 避免重复占用磁盘空间 |
| 复制 | 回退 | 不支持硬链接时使用 |

将 `raw mirror` 中的所有静态资源同步到 `localized mirror`：

#### 必须同步的资源清单

| 类别 | 资源 | 视觉影响 |
|------|------|---------|
| **样式** | CSS 文件 | 决定页面布局和外观 |
| **字体** | Web 字体文件 | 决定文字显示效果 |
| **图标** | 图标字体、SVG 图标 | 决定图标显示 |
| **图片** | 所有图片资源 | 决定图片显示 |
| **脚本** | JavaScript 文件 | 决定页面交互 |
| **配置** | manifest、json | 决定 PWA 行为 |

#### 同步验证

同步完成后需验证：
- [ ] CSS 文件完整同步
- [ ] 字体文件完整同步
- [ ] 图片资源完整同步
- [ ] JavaScript 文件完整同步
- [ ] 页面可正常渲染，无资源缺失

### 4. 状态流转

`queued` → `rebuilding` → `completed` / `failed`

### 5. 阶段 2 完成判定

- `localize_queue` 中所有 URL 状态为 `completed` 或 `failed`
- 所有静态资源已同步到 `localized mirror`
- 统计成功/失败数量，记录日志

---

## 预览适配

| 阶段 2 状态 | 预览行为 |
|------------|---------|
| 未完成 | 返回"已下载，等待本地化重建"占位页 |
| 已完成 | 优先读取 `localized mirror`，确保视觉与原站一致 |

## 增量更新流程

```
┌─────────────────────────────────────────────────────────────┐
│                    增量更新流程                              │
│                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│  │ 1.刷新  │ →  │ 2.变更  │ →  │ 3.增量  │ →  │ 4.翻译  │ │
│  │ 种子清单│    │ 检测    │    │ 执行    │    │ 记忆复用│ │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 1. 刷新种子清单

重新获取 sitemap / index.json

### 2. 变更检测

比较 `lastmod` / `etag` / `content_hash`

### 3. 增量执行

| 操作 | 说明 |
|------|------|
| 抓取 | 仅抓取新增或变更的页面/资源 |
| 抽取 | 仅重抽取变更页面的文本单元 |
| 导出 | 仅导出新增待翻译文本 |
| 重建 | 仅重建受影响页面与搜索索引 |

### 4. 翻译记忆复用

未变化文本不重新导出、不重新回填
