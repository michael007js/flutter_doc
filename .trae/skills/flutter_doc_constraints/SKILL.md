# Flutter/Dart 文档本地化工具 - 开发约束

## 名词约束
- **工程目录**：C:\app\flutter doc，本文且其他技能中提到的【工程目录】都是指本条说明中的目录，请勿改变。

## 关键约束

**重要关键**：

- 严格按照本文档开发，不改变任何功能。

## 工程约束
- **数据目录**：【工程目录】下的 `data` 目录，包含项目的数据库、翻译记忆、任务状态、日志等文件、镜像数据、本地化数据等所有数据文件。
- **配置文件**：【工程目录】下的 `flutter_doc_config.json` 文件，配置文件采用 JSON 格式，包含项目路径，重启后自动加载。

## 技术选型约束

- **开发语言**：保持 Python 桌面方案，不切换到 Flutter 桌面或 Electron。
- **Python 版本**：Python 3.8+，确保兼容性。
- **GUI 框架**：PySide6 6.5.x + QWebEngineView，不使用其他 GUI 框架。
- **数据库**：SQLite (启用 FTS5 全文搜索)，不使用其他数据库。
- **打包工具**：PyInstaller，不使用其他打包工具。

## 功能约束

- **资源本地化定义**：四站全部页面与其实际依赖的静态资源镜像到本地，且四站互相指向的同步镜像，外部导航页面不递归。
- **目标站点固定**：仅支持以下 4 个站点，不扩展到其他站点：
  1. `docs.flutter.dev`
  2. `api.flutter.dev`
  3. `dart.dev`
  4. `api.dart.dev`
- **镜像优先级**：首次全量镜像以稳定可恢复优先，不追求单次跑满带宽。
- **数据库迁移**：旧项目数据库支持安全迁移到新结构。
- **外部链接处理**：外域导航保留外链，不做本地化处理。

## 开发流程约束

- **确认机制**：为保证项目的顺利性，开始前需要与我确认项目计划，有不清楚或模糊的地方，不要自己假设，需要与我确认。
- **文档优先**：严格按照本文档开发，不改变任何功能，如有疑问先确认再开发。
- **增量开发**：每次开发完成后需要验证功能正确性，再进行下一步开发。

## 代码规范约束

- **代码注释**：为项目代码添加详细且专业的注释，说明每个模块、每个类、每个函数、每个变量等的功能和实现细节。
- **代码风格**：遵循 PEP 8 Python 代码风格规范。
- **命名规范**：
  - 类名使用大驼峰命名法（如 `MirrorService`）
  - 函数名和变量名使用小写下划线命名法（如 `mirror_site`）
  - 常量使用全大写下划线命名法（如 `MAX_RETRY_COUNT`）
- **文件编码**：所有 Python 文件使用 UTF-8 编码。

## 界面约束

- **界面语言**：GUI 中的用户交互界面必须是中文。
- **标签页数量**：GUI 保持 5 个主标签页，不增减。
- **CLI 命令**：CLI 命令保持现有列表，不随意添加新命令。

## 性能约束

- **并发控制**：使用双线程池分离（下载线程池和本地化重建线程池），两池互不干扰。
- **内存管理**：避免内存泄漏，及时释放不再使用的资源。
- **数据库连接**：使用单一数据库连接池，不再频繁开关数据库连接。
- **Web服务**：打开/关闭预览服务时，不可对主线程进行阻塞操作，保持 GUI 响应性。
- **控件响应**：所有 GUI 控件（如按钮、输入框、下拉菜单等）必须响应用户操作，保持交互性。
- **主线程阻塞**：所有耗时操作（如网络请求、数据库操作、文件操作等）必须在后台线程中进行，不可阻塞主线程。

## 安全约束

- **文件操作**：文件操作前检查路径有效性，避免路径遍历攻击。
- **输入验证**：对用户输入进行验证，防止注入攻击。

## 测试约束

- **测试覆盖**：核心功能必须有对应的测试用例。
- **测试类型**：包含单元测试、集成测试、任务控制测试、增量测试、打包测试。

## 打包约束

- **打包格式**：生成 Windows exe 文件，不生成其他平台版本。
- **文件结构**：脚本不能将所有文件全部打包到 exe 中，所有的必须文件需要存放到目录中。
- **编码问题**：编译脚本不能包含中文字符，避免在 Windows 上双击运行时出错。

## 版本控制约束

- **提交频率**：每完成一个功能模块提交一次代码。
- **提交信息**：提交信息使用中文，清晰描述本次提交的内容。
- **分支管理**：主分支保持稳定，新功能在开发分支开发完成后合并。

## 文档约束

- **README**：保持 README 文件的更新，说明项目使用方法。
- **技能文档**：每个技能模块的 SKILL.md 文件保持独立和完整。
- **注释同步**：代码注释与代码实现保持同步更新。
