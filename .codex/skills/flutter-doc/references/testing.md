# 测试计划

## 单元测试

覆盖以下底层规则：

- URL 规范化
- 四站互链改写
- 外域静态资源本地化规则
- CSS `url()` 改写
- 文本保护占位符
- hash 去重
- 翻译包导出/导入校验
- 翻译记忆复用

## 抽取规则测试

至少覆盖四类样本页：

- Flutter 文档页
- Dart 文档页
- Flutter API 页
- Dart API 页

关键断言：

- 说明文本会被抽出
- `signature` / `pre` / `code` / identifier 不被抽出

## 集成测试

- 四站互链本地化正确
- 必要外域静态资源本地可加载
- 外部导航保留外链
- 双阶段流程能完整执行
- 预览服务可访问

## 任务控制测试

- 开始后暂停
- 暂停后继续
- 程序退出后恢复
- 失败任务单独重试
- 多线程下不重复领取同一任务

## 增量测试

修改单页 `hash` 后执行 `update`，确认：

- 只重抓变化页及其依赖
- 只新增导出变更文本
- 只重建受影响索引

## 导出导入测试

执行完整链路：

1. 导出待翻译包
2. 人工模拟翻译
3. 导入并回填
4. 验证页面中文可见且占位符完好

## 打包测试

生成 `exe` 后，在 Windows 上验证：

- 双击启动
- 创建项目
- 执行镜像
- 暂停与恢复
- 导出与导入翻译包
- 本地预览站点
