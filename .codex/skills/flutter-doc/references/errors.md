# 常见错误与修复

## 1. SQLite 事务提交错误

问题：`INSERT OR IGNORE` 没有实际写入时仍然 `commit`，可能触发“no transaction is active”。

修复原则：

- 只在 `conn.in_transaction` 为真时提交或回滚

## 2. 无意义事务

问题：没有实际写操作也频繁 `commit`。

修复原则：

- 只在真正发生变更时提交

## 3. 跨线程信号问题

问题：工作线程直接触发 GUI 更新，导致跨线程 UI 问题。

修复原则：

- Qt 信号连接优先使用 `Qt.QueuedConnection`

## 4. SQLite 锁定

问题：多线程访问 SQLite 时出现 `database is locked`。

修复原则：

- 使用线程锁保护数据库操作
- 设置合理连接超时
- 启用 WAL

## 5. HTTP 超时未处理

问题：请求超时导致卡死、崩溃或假死。

修复原则：

- 设置超时
- 配置重试与退避
- 捕获超时异常并写日志

## 6. Windows 路径编码问题

问题：含空格或中文路径时，字符串拼接与转义容易出错。

修复原则：

- 优先使用 `Path`
- 避免手写带转义风险的路径字符串
- 保持 UTF-8

## 7. 信号未断开导致内存泄漏

问题：Worker 结束后信号连接残留，导致对象无法释放。

修复原则：

- 在线程完成时断开信号
- 清理 worker 生命周期

## 8. 大文件下载内存溢出

问题：一次性把整个响应读入内存。

修复原则：

- 使用流式下载
- 分块写盘

## 9. HTML 编码检测错误

问题：页面编码不一致，导致乱码或解析失败。

修复原则：

- 结合响应头、`meta` 与解析器编码检测
- 在解析前明确或校正编码

## 10. 线程池未正确关闭

问题：程序退出时线程池仍在运行，导致进程挂住。

修复原则：

- 使用上下文管理或显式 `shutdown`
- 在退出链路中保证资源回收

## 11. 项目配置保存看似成功但重启后失效

问题：GUI 点击保存后提示成功，但重新打开程序后配置没有生效，常见于打包版把配置写到了错误目录。

修复原则：

- 不要把默认配置路径直接绑定到 `Path(__file__)` 推导的源码目录
- 将应用根目录解析收敛到单一函数，例如 `resolve_app_root()`
- 源码模式使用工程根目录；冻结模式使用 `Path(sys.executable).parent`
- `load()` 和 `save()` 必须共用同一套默认路径解析逻辑
- 保存后返回实际落盘路径，并在 GUI 中直接展示

## 12. 点击“开始镜像”只有日志，没有真正启动任务

问题：按钮点击后只输出“开始镜像功能已准备好。”之类的占位文案，没有调用镜像服务。

修复原则：

- 禁止把核心按钮只接到 `append_log()` 之类的占位实现
- 使用 worker + `QThread` 在后台执行 `MirrorService`，避免阻塞 GUI
- GUI 只通过信号槽接收日志、完成态和错误信息
- “暂停/继续”必须作用于运行中的同一 `TaskScheduler`
- 双阶段状态从 SQLite 的 `task_status` 读取，避免界面显示和实际任务脱节
- 窗口关闭时要停止轮询并向 worker 发出协作式停止信号

## 13. 镜像进度显示成 `1/1 -> 2/2 -> 5/5` 这类同步增长的假进度

问题：抓取/更新页里“完成数”和“总数”几乎同时增加，看起来像任务一直是满进度推进，但实际只是 `total_count` 被错误地写成了 `completed + failed`。

修复原则：

- `task_status.total_count` 必须来自真实队列汇总，而不是运行时临时计数
- 下载阶段使用 `url_queue` 汇总 `queued/downloading/downloaded/failed`
- 本地化阶段使用 `localize_queue` 汇总 `queued/rebuilding/completed/failed`
- 阶段结束后根据剩余队列判断终态：无剩余为 `completed`，用户停止且仍有剩余为 `paused`，异常中断且仍有剩余为 `failed`
- GUI 读取 `task_status` 时，看到的 `完成/总数` 应能反映“已完成多少 / 当前已发现多少”，而不是“已完成多少 / 已完成多少”
