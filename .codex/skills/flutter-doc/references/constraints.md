# 开发约束

这个文档是所有任务的起点。开始设计或实现前，先锁定这些边界，避免做出与项目目标冲突的决定。

## 工程与基础边界

- **工程目录**固定为 `C:\app\flutter doc`。
- **数据目录**固定在工程目录下的 `data/`。
- **配置文件**固定为工程目录下的 `flutter_doc_config.json`。
- 提到“工程目录”时，默认都指上述路径。

## 技术选型边界

- 保持 **Python 3.8+** 桌面方案。
- GUI 使用 **PySide6 6.5.x + QWebEngineView**。
- 数据库使用 **SQLite + FTS5**。
- 打包使用 **PyInstaller**。
- 不切换到 Flutter Desktop、Electron 或其他数据库、GUI、打包方案。

## 产品与功能边界

- 只支持四个固定站点：
  - `docs.flutter.dev`
  - `api.flutter.dev`
  - `dart.dev`
  - `api.dart.dev`
- “资源本地化”指四站页面及其实际依赖的静态资源镜像到本地，并保证四站互链离线可用。
- 外域导航保留外链，不递归扩展成新站点。
- 首次全量镜像以稳定、可恢复优先，不追求极限带宽。
- 旧项目数据库需要支持安全迁移到新结构。

## 开发流程边界

- 按文档要求开发，不随意删减或改写既定功能目标。
- 有明显模糊处时先确认，不靠猜测推进。
- 采用增量开发：每完成一块功能都要验证，再进入下一块。

## 代码与实现约束

- Python 文件统一使用 UTF-8 编码。
- 遵循 PEP 8。
- 命名约定：
  - 类名：大驼峰，如 `MirrorService`
  - 函数/变量：下划线命名，如 `mirror_site`
  - 常量：全大写下划线，如 `MAX_RETRY_COUNT`
- 注释需要专业、同步、能解释模块职责与关键实现。

## GUI / CLI 约束

- GUI 面向中文用户。
- GUI 保持 **5 个主标签页**，不随意增减。
- CLI 保持既有命令集合，不任意扩容。
- 所有耗时操作必须放到后台线程，不能阻塞主线程。
- 预览服务启停不能阻塞 GUI。

## 并发与性能约束

- 使用双线程池分离下载与本地化重建。
- 数据库避免频繁开关连接，优先复用单一连接能力。
- 所有控件必须保持可交互。
- 注意资源释放，避免内存泄漏。

## 安全约束

- 文件操作前检查路径有效性，避免路径遍历。
- 用户输入要做校验，防止注入与非法路径。

## 测试与打包约束

- 核心功能必须有测试用例。
- 覆盖单元、集成、任务控制、增量、打包测试。
- 打包目标是 Windows `exe`。
- 打包脚本不能包含中文，避免双击运行编码问题。
- 不能把所有内容都塞进单一 `exe`，必要文件要保留在目录中。
