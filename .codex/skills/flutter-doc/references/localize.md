# 本地化服务

## 核心目标

- 四站互链本地化
- 视觉一致性
- 外部导航外链保留

本地化的目标不是简单替换链接，而是在离线环境里尽量保留原站浏览体验。

## 四站互链

需要让以下四站之间的内部引用都改写为本地路径：

- `docs.flutter.dev`
- `api.flutter.dev`
- `dart.dev`
- `api.dart.dev`

跨站互链仍然是“内部链接”，必须改写为本地镜像路径，而不是留在线访问。

## 外部链接处理

- 外部网站导航：保留原链接
- 外部 CDN 静态资源：如果页面渲染依赖它们，应下载并本地化
- 社交媒体、第三方服务链接：通常保留原链接

## CSS 处理

本地化时重点处理：

- CSS 文件中的 `url()`
- `@font-face` 字体引用
- `background-image` 等样式资源
- 内联 `<style>` 中的依赖资源

常见改写包括：

- 绝对路径改为相对路径
- 外域字体与图片资源下载后指向本地
- 已正确的相对路径保持不变

## 图片与媒体处理

重点覆盖：

- `<img src>`
- `<img srcset>`
- `<picture>` / `<source>`
- `<svg href>`
- `<video src>` / `poster`
- `<audio src>`
- CSS 中的图片引用

`srcset` 需要把其中所有候选路径都改写完整，不能只改默认 `src`。

## 资源同步

推荐同步策略：

1. 优先使用硬链接，减少重复占盘
2. 不支持时退回复制

同步对象包括：

- CSS
- 字体
- 图标字体
- 图片
- JS
- 配置类静态资源

## 视觉一致性检查

本地化完成后至少验证：

- 布局
- 字体
- 图标
- 图片
- 颜色
- 动画/过渡
- 响应式行为
